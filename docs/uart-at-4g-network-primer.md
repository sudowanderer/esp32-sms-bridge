# UART、AT 指令与 4G 模块网络能力科普

本文面向后续重构前的技术理解，解释 ESP32C3 如何通过 ML307R-DC 4G 模块收短信、发 AT 指令，以及如果要让设备走 4G 流量访问网络，应该怎么理解相关技术路线。

## ESP32C3 和 4G 模块分别负责什么

可以先把这套硬件理解成：

```text
ESP32C3 = 小电脑 / 控制器
ML307R-DC = 4G 手机模块
UART 串口 = 两者之间的聊天线
AT 指令 = ESP32C3 对 4G 模块说的话
```

ESP32C3 本身不会接入移动网络，也不会直接收短信。真正负责蜂窝网络、SIM 卡、短信、4G 数据连接的是 ML307R-DC。

ESP32C3 的职责是：

- 通过串口控制 ML307R-DC。
- 配置短信模式。
- 读取 ML307R-DC 上报的短信数据。
- 解码短信内容。
- 再通过 WiFi 或 4G 模块网络能力把短信转发出去。

## RX / TX 是什么

RX 和 TX 是串口通信里的两个方向：

```text
TX = Transmit，发送
RX = Receive，接收
```

所以连接时要交叉：

```text
ESP32 TX  ->  ML307R RX
ESP32 RX  <-  ML307R TX
```

可以把它类比成人说话：

```text
你的嘴巴  -> 对方耳朵
你的耳朵  <- 对方嘴巴
```

如果 TX 接 TX、RX 接 RX，就类似嘴巴对嘴巴、耳朵对耳朵，两边都听不到对方。

当前项目的引脚定义是：

```cpp
#define TXD 3
#define RXD 4
```

含义是：

```text
ESP32C3 GPIO3 作为 TX，接 ML307R RX
ESP32C3 GPIO4 作为 RX，接 ML307R TX
```

## 波特率是什么

波特率可以简单理解成串口的“说话速度”。

当前代码中：

```cpp
Serial1.begin(115200, SERIAL_8N1, RXD, TXD);
```

其中 `115200` 就是波特率。

你不需要深入理解电子层面的细节，只要记住一点：

```text
ESP32C3 和 ML307R-DC 的波特率必须一致。
```

如果 ESP32C3 用 `115200`，ML307R-DC 也用 `115200`，双方就能正常通信。

如果一边是 `115200`，另一边是 `9600`，就会乱码，类似两个人用完全不同的语速和节奏说话。

常见波特率：

```text
9600
115200
921600
```

这个项目使用 `115200`，这是 AT 模块里很常见、也比较稳妥的速度。

## SERIAL_8N1 是什么

这段代码：

```cpp
SERIAL_8N1
```

表示串口数据格式：

```text
8 = 每个字符 8 个数据位
N = No parity，没有校验位
1 = 1 个停止位
```

可以把它理解成双方约定的“说话格式”。

大多数 AT 模块默认都是 `8N1`，所以实际开发中通常直接使用：

```cpp
Serial1.begin(115200, SERIAL_8N1, RXD, TXD);
```

这行代码翻译成人话就是：

```text
使用 Serial1 串口；
通信速度是 115200；
数据格式是 8N1；
ESP32 的 RX 是 GPIO4；
ESP32 的 TX 是 GPIO3；
开始和 4G 模块通信。
```

## UART 串口怎么理解

UART 是一种非常常见的串行通信方式。

它的特点是：

- 线少，通常只需要 TX、RX、GND。
- 双方按约定波特率传输字节。
- 收到的数据本质上是一串字符或字节。
- 很适合 MCU 和通信模块之间交互。

在这个项目里，UART 上会出现这些内容：

```text
ESP32C3 -> ML307R: AT
ML307R -> ESP32C3: OK

ESP32C3 -> ML307R: AT+CEREG?
ML307R -> ESP32C3: +CEREG: 0,1
ML307R -> ESP32C3: OK

ML307R -> ESP32C3: +CMT: ...
ML307R -> ESP32C3: 089168...
```

所以 UART 不只是命令通道，也是短信上报通道。

## AT 指令是什么

AT 指令本质上是一套文本命令。

ESP32C3 通过 UART 给 ML307R-DC 发送字符串：

```text
AT
```

模块返回：

```text
OK
```

这表示模块在线，并且能听懂命令。

可以把 ML307R-DC 想成一个“没有屏幕的 4G 手机”，ESP32C3 通过 AT 指令控制它。

例子：

```text
ESP32C3: AT
ML307R:  OK

ESP32C3: AT+CESQ
ML307R:  +CESQ: 99,99,255,255,18,60
ML307R:  OK
```

这有点像在命令行里输入命令，只不过命令不是发给电脑操作系统，而是发给 4G 模块。

## 为什么叫 AT 指令

这是历史原因。

早期调制解调器命令以 `AT` 开头，来自 “attention”，可以理解成“注意，我要发命令了”。

所以很多模块命令长这样：

```text
AT
ATI
AT+CMGF=0
AT+CNMI=2,2,0,0,0
AT+CEREG?
AT+CESQ
```

## AT 指令能控制哪些能力

4G 模块的 AT 指令能力很多，具体取决于模块厂商和 AT 固件。

常见能力包括：

| 能力 | 示例指令 | 说明 |
|---|---|---|
| 测试模块是否在线 | `AT` | 返回 `OK` 表示模块可通信 |
| 查询模块信息 | `ATI` | 查询厂家、型号、固件版本 |
| 查询 SIM 信息 | `AT+CIMI` / `AT+ICCID` | 查询 IMSI、ICCID |
| 查询信号 | `AT+CESQ` | 获取 LTE 信号质量 |
| 查询网络注册 | `AT+CEREG?` | 判断是否已注册移动网络 |
| 查询运营商 | `AT+COPS?` | 获取当前运营商 |
| 设置短信模式 | `AT+CMGF=0` | 设置 PDU 短信模式 |
| 设置短信上报 | `AT+CNMI=2,2,0,0,0` | 新短信直接通过串口上报 |
| 发送短信 | `AT+CMGS=...` | PDU 模式发送短信 |
| 控制数据连接 | `AT+CGACT=1,1` | 激活 PDP 数据连接 |
| Ping 测试 | `AT+MPING="8.8.8.8",30,1` | 让模块通过 4G 网络 ping |
| 功能模式 | `AT+CFUN?` / `AT+CFUN=1` / `AT+CFUN=4` | 查询/设置全功能或飞行模式 |

有些 4G 模块还支持：

- TCP socket
- UDP socket
- HTTP client
- HTTPS client
- MQTT client
- DNS
- SSL/TLS
- NTP 时间同步

但这些能力是否存在、具体命令是什么，要查 ML307R 的 AT 指令手册。

## 短信为什么是 PDU

短信模块收到的原始短信不一定直接是可读文本。

在 PDU 模式下，模块上报的短信可能类似：

```text
0891683108200505F0240D9168...
```

这串十六进制内容里包含：

- 短信中心号码
- 发送者号码
- 时间戳
- 编码方式
- 短信正文
- 长短信分片信息

所以代码需要用 PDU 库解码：

```cpp
pdu.decodePDU(...)
pdu.getSender()
pdu.getTimeStamp()
pdu.getText()
pdu.getConcatInfo()
```

解码后才能得到：

```text
发送者是谁
什么时候发的
短信内容是什么
是不是长短信的一部分
```

## 短信接收完整流程

当前项目的短信接收流程可以理解为：

```text
1. ESP32C3 上电
2. ESP32C3 拉 GPIO5，重启 ML307R
3. ESP32C3 通过 UART 发送 AT
4. ML307R 返回 OK
5. ESP32C3 设置短信模式：
   AT+CMGF=0
   AT+CNMI=2,2,0,0,0
6. ML307R 注册移动网络
7. SIM 卡收到短信
8. ML307R 通过 UART 上报：
   +CMT:
   PDU 数据
9. ESP32C3 解码 PDU
10. ESP32C3 通过 WiFi 或 4G 模块网络能力转发短信
```

关键点：

```text
ESP32C3 不直接收短信。
ML307R 收短信。
ESP32C3 读取 ML307R 从串口吐出来的数据。
```

## ESP32C3 通过 4G 模块访问网络的三种方式

如果想让设备不用 WiFi，而是通过 ML307R 的 4G 流量访问网络，主要有三种路线。

### 路线一：AT HTTP / TCP

这是最适合本项目优先考虑的方式。

ESP32C3 仍然通过 UART 发 AT 指令，但这次命令 ML307R 自己联网、自己发 HTTP/TCP。

可以理解成：

```text
ESP32C3 -> ML307R: 帮我打开数据连接
ESP32C3 -> ML307R: 帮我 POST 到服务器
ML307R -> 互联网: 通过 4G 流量请求服务器
ML307R -> ESP32C3: 返回请求结果
```

优点：

- 对 ESP32C3 内存压力相对小。
- 不需要让 ESP32C3 自己跑完整 4G 网络协议栈。
- 适合“收到短信后发一个 HTTP 请求”的场景。

缺点：

- 能力受 ML307R AT 固件限制。
- HTTPS、证书、重定向、长连接等功能可能比较麻烦。
- 所有流程都要通过 AT 状态机管理。

建议：

```text
如果后续要支持 SIM 卡流量转发短信，优先研究 ML307R 的 AT HTTP/TCP 指令。
```

### 路线二：PPP 拨号

PPP 全称是 Point-to-Point Protocol。

可以简单理解成：

```text
ESP32C3 通过串口和 ML307R 建立一条网络链路；
之后 ESP32C3 自己获得一个 IP；
ESP32C3 自己使用 TCP/IP、HTTP、MQTT、TLS。
```

大概流程：

```text
ESP32C3 -> ML307R: AT 初始化
ESP32C3 -> ML307R: 拨号
进入 PPP 数据模式
ESP32C3 的网络栈接管串口
ESP32C3 获得 4G 网络 IP
ESP32C3 自己发 HTTP 请求
```

优点：

- ESP32C3 可以像使用 WiFi 一样使用 TCP/IP。
- HTTP、MQTT、TLS 逻辑更统一。

缺点：

- 实现复杂。
- PPP 会占用同一个 UART。
- PPP 数据模式下，短信 URC 和普通 AT 命令管理更麻烦。
- ESP32C3 内存压力更大。
- 串口 `115200` 带宽有限，网速很慢。
- 对“稳定收短信”目标来说风险更高。

建议：

```text
第一版稳定固件不建议使用 PPP。
```

### 路线三：USB 网卡模式

有些 4G 模块可以通过 USB 暴露成网卡，例如 RNDIS、ECM、NCM。

这种方式类似电脑插 USB 4G 网卡。

但当前项目不推荐优先考虑：

- 当前硬件是 ESP32C3 和 ML307R 通过 UART 连接。
- ESP32C3 做 USB Host 驱动 4G 网卡复杂度很高。
- 不适合这个低成本短信转发项目。

建议：

```text
当前硬件和目标下，基本不用考虑 USB 网卡模式。
```

## 为什么重构时 Serial1 必须统一管理

UART 是一条共享通道。

这条线上会同时出现：

```text
AT 响应
短信上报
Ping 结果
网络状态
错误信息
```

如果多个模块都直接读 `Serial1`，就容易混乱。

比如短信刚到：

```text
+CMT: ...
089168...
```

此时 Web 页面刚好执行信号查询，代码如果先清空串口：

```cpp
while (Serial1.available()) Serial1.read();
```

短信就可能被直接丢掉。

因此重构时必须遵守：

```text
只有 modem_at 模块能读取 Serial1。
其他模块都向 modem_at 提交请求。
```

这相当于公司只有一个前台接电话，其他部门不能抢电话线。

推荐模型：

```text
Web 页面
短信发送模块
Ping 模块
状态查询模块
        |
        v
    modem_at
        |
        v
     Serial1
        |
        v
     ML307R
```

## 给本项目的技术建议

如果目标是长期稳定短信转发，建议优先级如下：

1. 先把 UART / AT 调度器写稳。
2. 让 `modem_at` 独占 `Serial1`。
3. 短信接收、AT 响应、URC 都由同一个串口解析器处理。
4. 收到短信后先入队，不要直接同步推送。
5. 网络转发失败不能影响继续收短信。
6. 第一版继续使用 WiFi HTTP 推送。
7. 后续如果要走 SIM 卡流量，优先使用 ML307R 的 AT HTTP/TCP 能力。
8. 不建议第一版上 PPP。
9. 不建议考虑 USB 网卡模式。

一句话总结：

```text
ESP32C3 用 UART 和 ML307R 说话；
TX 是发送，RX 是接收；
波特率是说话速度；
AT 指令是控制 4G 模块的文本命令；
ML307R 负责真正的 4G 网络和短信；
ESP32C3 负责配置模块、监听短信、解码短信、转发短信。
```
